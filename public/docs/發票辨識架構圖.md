## 💡 六、技術亮點說明

### 6.1 設計模式應用

| 模式 | 應用位置 | 效益 |
|------|---------|------|
| **Service Layer** | `len/services/` (25+ 服務模組) | 業務邏輯與視圖分離，提升可測試性 |
| **Custom Manager** | `GoogleInvoiceQuerySet` | 封裝複雜查詢，提升程式碼重用性 |
| **Repository Pattern** | ORM QuerySet 封裝 | 資料存取邏輯集中管理 |
| **Strategy Pattern** | 報表生成服務 | 不同報表類型使用不同生成策略 |

### 6.2 效能優化技術

```python
# 使用 select_related 避免 N+1 查詢
queryset.select_related('AccountingCategory', 'Image', 'Image__user')

# 使用 annotate 在資料庫層計算
.annotate(
    Debit=Sum(Case(When(InvoiceClassify='expense', then='TotalAmount'))),
    Credit=Sum(Case(When(InvoiceClassify='income', then='TotalAmount')))
)

# 條件聚合查詢
.values('AccountingCategory__parent__name')
.annotate(expense=Sum(Case(...)))
```

### 6.3 前端技術整合

- **模組化 JavaScript**：將 500+ 行程式碼重構為獨立模組
- **AJAX 非同步處理**：提升使用者體驗
- **WebRTC 整合**：瀏覽器直接存取攝影機
- **jqWidgets 元件**：進階 UI 互動

## 📝 面試時的說明重點

### 當面試官問「這個系統做什麼？」

> 這是一個**企業會計自動化系統**，核心功能是透過 **OCR 影像辨識**將紙本發票轉換為結構化會計資料，自動產生傳票並生成財務報表（資產負債表、損益表等）。

### 當面試官問「技術架構如何設計？」

> 採用 **Django 分層架構**：
> 1. **Presentation Layer**：CBV 處理 HTTP 請求
> 2. **Business Logic Layer**：25+ 個 Service 模組處理業務邏輯
> 3. **Data Access Layer**：Custom QuerySet Manager 封裝查詢
> 4. **Database Layer**：關聯式資料庫儲存
>
> 這樣的設計讓業務邏輯與框架解耦，提升可測試性和可維護性。

### 當面試官問「有什麼技術亮點？」

> 1. **Service Layer 設計**：業務邏輯完全獨立於 Views，符合 SOLID 原則
> 2. **Custom QuerySet Manager**：封裝複雜的 ORM 查詢，避免程式碼重複
> 3. **效能優化**：使用 select_related、annotate 等技術避免 N+1 問題
> 4. **前端模組化**：將大型 JavaScript 檔案重構為可維護的模組
> 5. **多租戶設計**：支援多公司資料隔離
