## 💡 五、技術亮點說明

### 5.1 設計模式應用

| 模式 | 應用位置 | 效益 |
|------|---------|------|
| **Service Layer** | `schedule_generator.py`、`holiday_service.py` | 業務邏輯與視圖分離,提升可測試性 |
| **Mixin Pattern** | `BossRequiredMixin`、`EmployeeSelectorMixin` | 權限控制與功能共用,符合 DRY 原則 |
| **Signal Observer** | `post_save(Company) → sync_holidays()` | 自動化流程,減少手動操作 |
| **Strategy Pattern** | 排班演算法(全日班 vs 時段班) | 不同員工類型使用不同排班策略 |

### 5.2 效能優化技術

```python
# 使用 select_related 避免 N+1 查詢
Employee.objects.select_related('company').filter(...)

# 批次建立班次
Shift.objects.bulk_create([
    Shift(employee=emp, date=date, ...)
    for emp, date in assignments
])

# 資料庫層聚合計算
LeaveRequest.objects.filter(
    employee=user, status='approved'
).aggregate(total=Count('id'))
```

### 5.3 即時通知架構

```
員工提交請假
    ↓
Django View 處理
    ↓
儲存資料庫
    ↓
發送 WebSocket 訊息
    ↓
channel_layer.group_send('boss', {...})
    ↓
老闆端即時收到通知
```

**技術選型**:
- 開發環境: InMemoryChannelLayer (Windows/macOS)
- 生產環境: RedisChannelLayer (Linux/Docker)

## 📝 八、面試時的說明重點

### 當面試官問「這個系統做什麼?」

> 這是一個**企業自動排班系統**,核心功能是讓員工線上提交請假申請,老闆審核後系統會自動產生最佳班表。系統會根據員工等級、請假記錄、人力需求等條件,使用智能演算法自動分配班次,減少人工排班時間 80% 以上。

### 當面試官問「技術架構如何設計?」

> 採用 **Django 分層架構**:
> 1. **Presentation Layer**: CBV 處理 HTTP 請求,WebSocket 處理即時通知
> 2. **Business Logic Layer**: Service Layer 封裝排班演算法、假日同步等核心邏輯
> 3. **Data Access Layer**: Django ORM + Signals 處理資料存取與自動化流程
> 4. **Database Layer**: 多租戶資料隔離設計
>
> 這樣的設計讓業務邏輯與框架解耦,提升可測試性和可維護性。

### 當面試官問「有什麼技術亮點?」

> 1. **排班演算法引擎**: 自動依員工等級加權分配,滿足人力需求約束,區分全日班/時段班
> 2. **多租戶架構**: 支援多公司使用同一系統,資料完全隔離,可做 SaaS 平台
> 3. **即時通知系統**: 使用 Django Channels + WebSocket 實現雙向即時通訊
> 4. **假日自動同步**: Django Signals 自動觸發,從政府 API 同步假日並排除補班日
> 5. **Service Layer 設計**: 業務邏輯完全獨立於 Views,符合 SOLID 原則
> 6. **效能優化**: 使用 select_related、bulk_create 等技術避免 N+1 問題

### 當面試官問「為什麼要做這個系統?」

> 許多中小企業(如餐飲業、零售業)有大量正職+工讀生的混合排班需求,人工排班非常耗時且容易出錯。這個系統可以:
> - 自動化排班流程,節省管理時間
> - 確保公平性,避免人為偏誤
> - 即時通知機制,提升溝通效率
> - 多租戶設計,一套系統服務多間公司

### 當面試官問「遇到什麼技術挑戰?」

> 1. **排班演算法設計**: 要同時考慮員工等級、請假、人力需求、工讀生時段等多重約束,使用加權分配策略解決
> 2. **多租戶資料隔離**: 確保不同公司的資料完全隔離,在 QuerySet 層面統一加上公司過濾條件
> 3. **即時通知跨平台**: Windows 開發環境用 InMemory,Linux 生產環境用 Redis,自動偵測平台切換
> 4. **假日資料同步**: 政府 API 包含「補班日」,需要過濾 `isHoliday=false` 的資料
