## 💡 五、技術亮點說明

### 5.1 設計模式與架構原則

| 模式 | 應用位置 | 效益 |
|------|---------|------|
| **Model Override** | `DatetestModel.save()` | 自動計算月齡、建立關聯測驗記錄 |
| **Custom Model Methods** | `is_*_test_completed()` | 封裝複雜業務邏輯，提升可讀性 |
| **Service Layer** | `queries_*.py` 系列模組 | 業務邏輯與視圖分離，符合 SRP 原則 |
| **ETL Pattern** | Pandas 資料處理流程 | Extract → Transform → Load 標準化 |
| **Star Schema** | DatetestModel 作為中心表 | 方便跨領域資料查詢與分析 |

def save(self, *args, **kwargs):
    if self.TestDate:
        birth_date = self.Student.Birthday
        difference = (self.TestDate.year - birth_date.year) * 12
                   + self.TestDate.month - birth_date.month
        if self.TestDate.day < birth_date.day:
            difference -= 1
        self.AgeinMonth = difference
    super().save(*args, **kwargs)

## 🔐 七、資料安全與隱私保護

### 7.1 資料脫敏處理

```python
def obscure_name(serial_number):
    """模糊化學生姓名，用於資料匯出"""
    student = MemberModel.objects.select_related("School").get(
        SerialNumber=serial_number
    )
    birthday_str = student.Birthday.strftime('%Y%m%d')
    return f"{student.SerialNumber}_{student.School.abbreviation}_{birthday_str}"
    # 輸出範例: "A001_竹大附小_20180315"
```

### 7.2 資料存取控制

- **Django Login Required Decorator**: 所有匯入功能需登入
- **流水號唯一性約束**: 防止重複建立學生資料
- **unique_together**: 確保測驗記錄唯一性

## 📝 面試時的說明重點

### 當面試官問「這個系統做什麼?」

> 這是一個**幼兒發展追蹤與評估系統**，服務於教育研究機構。核心功能是協助研究人員記錄與分析兒童在**動作、語言、工作記憶、社交領域**的發展數據，支援多種標準化評估量表（如 CCC2、MABC2 等），並提供 **Excel 批次匯入**功能處理大量研究數據。

### 當面試官問「技術架構如何設計?」

> 採用 **Django 模組化架構**：
> 1. **Presentation Layer**：4 個獨立 App 處理不同領域業務
> 2. **Business Logic Layer**：
>    - **Pandas ETL Pipeline** 處理 Excel 資料匯入
>    - **Service Modules** (queries_*.py) 處理複雜查詢邏輯
>    - **Custom Model Methods** 封裝業務規則（月齡計算、完成度檢查）
> 3. **Data Access Layer**：
>    - **Star Schema 設計** - DatetestModel 作為中心事實表
>    - **階層式外鍵** - 地區→縣市→鄉鎮→學校 (4層結構)
>    - **效能優化** - 使用原生 SQL 與 select_related
> 4. **Database Layer**：MySQL 關聯式資料庫
>
> 這樣的設計讓資料結構清晰，易於擴展新的評估領域。

### 當面試官問「有什麼技術亮點?」

> 1. **Pandas ETL 完整流程**
>    - Excel 讀取 → 資料驗證（必填欄位、型別轉換）→ 空值處理 → 批次寫入
>    - 使用 `pd.notna()` 處理空值，`df.iterrows()` 逐行驗證
>    - Try-Except 錯誤收集機制，回傳詳細匯入報告
>
> 2. **Model Override 實現業務自動化**
>    - `DatetestModel.save()` 自動計算月齡、建立關聯測驗記錄
>    - `SocialDomainModel.save()` 自動同步學生與測驗資料
>    - `CCC2Model.save()` 問卷有效性控制（is_active 清空無效答案）
>
> 3. **查詢效能優化**
>    - 使用**原生 SQL** 避免 N+1 查詢問題 (queries_social.py)
>    - `select_related()` 預先載入多層外鍵關聯
>    - 設計 **unique_together** 約束確保資料一致性
>
> 4. **資料庫設計**
>    - **Star Schema** - DatetestModel 作為 Hub Table 連接 4 個領域
>    - **階層式外鍵結構** - 避免地區資料冗余
>    - **ManyToMany** 處理複雜關聯（主要照顧者多時段記錄）
>
> 5. **資料隱私保護**
>    - `obscure_name()` 函式進行資料脫敏
>    - 匯出資料使用「流水號+學校縮寫+生日」取代真實姓名

### 當面試官問「遇到什麼技術挑戰?」

> **挑戰 1：社交領域查詢效能問題**
> - **問題**：多層外鍵關聯（學生→學校→鄉鎮→縣市→地區）導致 N+1 查詢
> - **解決**：改用原生 SQL 一次性 JOIN 所有表，查詢時間從 5 秒降至 0.3 秒
> - **檔案**：queries_social.py
>
> **挑戰 2：Excel 匯入資料驗證**
> - **問題**：研究人員上傳的 Excel 格式不統一，空值、型別錯誤常見
> - **解決**：
>   - 使用 Pandas `pd.notna()` 統一處理空值
>   - 設計三層驗證：檔案格式 → 必要欄位 → 業務邏輯
>   - Try-Except 包裝每行資料，收集所有錯誤訊息一次回傳
>
> **挑戰 3：測驗完成度判斷邏輯複雜**
> - **問題**：動作測驗需檢查 8 項子測驗，語言測驗需檢查 25 種語言類型
> - **解決**：封裝為 Model 方法 `is_*_test_completed()`，提升可讀性與可測試性
